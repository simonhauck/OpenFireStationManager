## Overview

React 19 + TypeScript single-page application built with Vite. Uses TanStack Router for
file-based routing with automatic code-splitting, Tailwind CSS v4 + DaisyUI v5 for styling,
and ships as a PWA via `vite-plugin-pwa`.

**Tech stack:**
- TypeScript ~5.9 (strict mode), React 19, Vite 7
- TanStack Router v1 — file-based routing, `autoCodeSplitting: true`
- Tailwind CSS v4 (via `@tailwindcss/vite` plugin — no `tailwind.config.*` needed) + DaisyUI v5
- `vite-plugin-pwa` + Workbox — service worker with hourly update checks
- npm (uses `package-lock.json`)

All commands below are run from the `client/` directory.

## Project Structure

```
src/
├── main.tsx                  ← app entry point (router setup, React root)
├── index.css                 ← global styles (Tailwind + DaisyUI imports only)
├── PWABadge.tsx              ← PWA install/update toast (standalone utility component)
├── routeTree.gen.ts          ← AUTO-GENERATED by TanStack Router — never edit manually
├── vite-env.d.ts             ← Vite env type declarations
├── assets/                   ← static assets imported by source files
└── routes/
    ├── __root.tsx            ← root layout (nav, <Outlet>, PWABadge, devtools)
    ├── index.tsx             ← "/" route
    └── about.tsx             ← "/about" route
```

Add new routes as files under `src/routes/`. TanStack Router picks them up automatically
and regenerates `routeTree.gen.ts` on the next `dev` or `build` run.

Organise new non-route source files by feature (e.g. `src/stations/`, `src/vehicles/`)
rather than by type (no `src/components/`, `src/hooks/` mega-folders).

## TypeScript

All compiler options live in `tsconfig.app.json` (covers `src/`) and `tsconfig.node.json`
(covers `vite.config.ts` and other tooling files).

**Active strict flags:**
- `"strict": true` — enables all strict type checks
- `"noUnusedLocals": true` / `"noUnusedParameters": true` — unused code is a compile error
- `"noFallthroughCasesInSwitch": true`
- `"noUncheckedSideEffectImports": true`
- `"erasableSyntaxOnly": true` — disallows `enum` and `namespace` (use `const` objects instead)
- `"verbatimModuleSyntax": true` — requires `import type` for type-only imports

**Rules:**
- Never use `any`. Use explicit types, generics, or `unknown` with narrowing.
- Use `import type` for every type-only import — the compiler enforces this:
  ```ts
  import type { Station } from "./types";
  ```
- Prefer inline `type` aliases over `interface` for local shapes; use `interface` only when
  declaration merging or `extends` is needed.
- Use non-null assertion (`!`) only where presence is structurally guaranteed (e.g. a DOM
  element that is always in the HTML). Document why with a comment if non-obvious.
- Do not use `enum`. Use `const` objects + `typeof` utility types instead:
  ```ts
  const Status = { Active: "active", Inactive: "inactive" } as const;
  type Status = (typeof Status)[keyof typeof Status];
  ```

## Code Style

### Formatting — Prettier (`client/.prettierrc.json`)

| Setting | Value |
|---|---|
| `printWidth` | 100 |
| `tabWidth` | 2 |
| `useTabs` | false |
| `semi` | true |
| `singleQuote` | false (double quotes) |
| `jsxSingleQuote` | false |
| `trailingComma` | `"es5"` |
| `arrowParens` | `"always"` |
| `endOfLine` | `"lf"` |

Run `npm run format` to auto-fix. CI uses `npm run format:check`.

### Linting — ESLint v9 (`eslint.config.js`)

Flat config format. Applies to `**/*.{ts,tsx}`, ignores `dist/`:
- `@eslint/js` recommended
- `typescript-eslint` recommended
- `eslint-plugin-react-hooks` recommended-latest (enforces Rules of Hooks)
- `eslint-plugin-react-refresh` vite preset (HMR safety)

Run `npm run lint` to auto-fix. Run `npm run lint:check` for a read-only check.

### Naming Conventions

| Element | Convention | Example |
|---|---|---|
| React component files | `PascalCase.tsx` | `StationCard.tsx` |
| Non-component TS files | `camelCase.ts` | `useStations.ts` |
| TanStack Router specials | `__` prefix | `__root.tsx` |
| Route files | `kebab-case.tsx` matching URL segment | `station-detail.tsx` |
| React component functions | `PascalCase` | `StationCard` |
| Variables and functions | `camelCase` | `fetchStation` |
| Route exports | named `Route` constant | `export const Route = ...` |

### Imports

Follow this group order (no enforced sorter — maintain manually):
1. External packages (`react`, `@tanstack/react-router`, etc.)
2. Absolute internal imports (`../` paths that cross feature boundaries)
3. Relative imports (`./`)
4. CSS / asset imports last

Always use `import type` for type-only imports (enforced by `verbatimModuleSyntax`).

### Styling

- Use Tailwind utility classes for all layout and styling.
- Use DaisyUI component classes (`btn`, `card`, `modal`, etc.) for UI components.
- Avoid inline `style` props except for values that cannot be expressed as utilities
  (e.g. dynamic numeric values). `PWABadge.tsx` uses inline styles as a deliberate
  exception because it is a self-contained utility with no Tailwind dependency.
- Do not write custom CSS in `index.css` beyond the two framework imports and bare resets.

### Error Handling

- Use guard clauses with early returns; avoid deeply nested conditionals:
  ```ts
  async function save(data: FormData) {
    if (!data.name) return;
    // ...
  }
  ```
- No bare `catch (e) {}` — handle the error or re-throw it with added context.
- For async operations in event handlers, surface errors to the user (toast / error state);
  do not silently swallow them.

### Comments

- Inline `//` for brief clarifications.
- JSDoc `/** ... */` for exported utility functions with non-obvious behaviour.
- No comments that restate what the code already clearly expresses.

## Routing (TanStack Router)

- Add routes by creating files under `src/routes/`. The file path maps directly to the URL:
  `src/routes/stations/index.tsx` → `/stations`
  `src/routes/stations/$id.tsx` → `/stations/:id`
- Every route file must export a named `Route` constant created with `createFileRoute`.
- `src/routeTree.gen.ts` is **auto-generated** — never edit it. It regenerates on `dev`/`build`.
- `__root.tsx` is the root layout; add global UI (nav, footer, providers) there.
- Use `<Link>` from `@tanstack/react-router` for navigation; never use plain `<a>` tags for
  internal links.
- Use TanStack Router's `loader` for data fetching tied to a route; avoid fetching in
  `useEffect` for route-level data.

## PWA

- `PWABadge.tsx` handles both the install prompt and the "new version available" refresh toast.
  It is mounted once in `__root.tsx` and requires no props.
- The service worker registers with `registerType: "autoUpdate"` and checks for updates
  every hour via `registerPeriodicSync` inside `PWABadge.tsx`.
- PWA assets (icons, splash screens) are generated from `public/favicon.svg` using
  `pwa-assets.config.ts`. Run `npx vite-pwa/assets-generator` if the SVG changes.
- `devOptions.enabled` is `false` — the service worker does not run in `dev` mode.
